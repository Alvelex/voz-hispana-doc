# Sistemas

Cada sistema se documenta como fachada: qué hace, dónde está y cómo se usa.

No se explicará toda la implementación interna salvo que sea necesario para trabajar con seguridad.

Formato recomendado para cada sistema:

```text
1. Para qué sirve
2. Dónde está
3. API pública / cómo usarlo
4. Ejemplos mínimos
5. Reglas de equipo
6. Errores comunes
```
