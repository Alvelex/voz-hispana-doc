# Introducción

Esta documentación resume lo que un desarrollador nuevo debe saber para trabajar en el proyecto sin romper el flujo de carga, templates ni sistemas compartidos.

La regla general es simple: esta guía explica **cómo usar** cada sistema, no cómo está implementado por dentro. Los detalles internos solo se documentan cuando son necesarios para evitar errores comunes.

## Cómo leer esta documentación

Empieza por:

1. [Onboarding](./onboarding.md)
2. [Templates](./templates.md)
3. [Scripts](./scripts.md)
4. [Flujo de arranque](./startup-flow.md)

Después consulta los módulos o sistemas concretos según lo que estés tocando.
